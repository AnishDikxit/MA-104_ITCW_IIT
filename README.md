# 🐍 Lab 1: Welcome to Python & The Open Source Workflow

Welcome to your first Python lab! Today, we are moving away from local installations and jumping straight into the cloud using **Google Colab**. 

We are also simulating a real-world software engineering environment. You will use the industry standard **"Fork and Pull Request"** workflow.

## 🎯 Lab Objectives
1. Fork a repository to create your own copy of the lab.
2. Link Google Colab to your GitHub account.
3. Write your first Python script.
4. Save your work to a new branch.
5. Submit a Pull Request (PR).

---

## 🛠️ Step-by-Step Instructions

### Step 1: Fork This Repository
You do not have permission to edit this TA repository directly. You need to make your own "photocopy."
1. Look at the top-right corner of this page.
2. Click the **Fork** button.
3. Leave the settings as default and click **Create Fork**.
4. *Verify:* Look at the top left of the page. It should now say `YourUsername / MA-104_ITCW_IIT
`. You are now the admin of this copy!

### Step 2: Open the Starter Code in Google Colab
We have provided a starter notebook with instructions for your Python code. 
1. Click the magical badge below to open the notebook directly in Google Colab.

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/AnishDikxit/MA-104_ITCW_IIT/blob/main/Lab1_Python.ipynb)

*(Note: Sign in with your university Google account if prompted).*

### Step 3: Write Your Code!
Follow the instructions inside the Colab Notebook to build your 10-line Number Guessing Game. Test it by clicking the "Play" button next to your code cells.

### Step 4: Save to GitHub (On a New Branch)
Once your code is working perfectly, it's time to save it back to your GitHub repository.
1. In Google Colab, go to the top menu and click **File** -> **Save a copy in GitHub**.
2. A pop-up will appear asking you to authorize Colab. Click authorize.
3. In the "Save to GitHub" dialog box, fill out the following:
   * **Repository:** Make sure this is set to *your* forked repo (e.g., `YourUsername/MA-104_ITCW_IIT
`).
   * **Branch:** Do not use `main`. Instead, type a new branch name using this format: `lab1/YOUR-ROLL-NUMBER` (e.g., `lab1/104`). Colab will automatically create this branch for you!
   * **File path:** Leave as is (e.g., `Lab1_Starter.ipynb`).
   * **Commit message:** "Completed Casino game logic".
4. Click **OK**. Colab will save the file and open your GitHub repository in a new tab.

### Step 5: Submit the Pull Request (PR)
You have saved the code to your personal fork, but now you need to submit it to the TA's original repository.
1. Go to the homepage of *your* forked repository on GitHub.
2. You should see a yellow banner at the top saying your branch had recent pushes. Click the green **Compare & pull request** button.
3. **Title:** Change the title to `[Your Name] - [Your Roll Number]`.
4. **Leave a comment**
5. Click **Create pull request**.

🎉 **Congratulations! You have officially submitted your first Python lab using the Open Source workflow!**
